🌐 Web Basics

Strong basics = powerful exploits. Learn the web, see the bugs.

🔄 The Web Lifecycle

You → type site.com → DNS resolves → HTTP request → server responds → page loads
  • Client: Your browser (Chrome, Firefox)
    🐞 Bug bounty angle: Understand how requests originate to intercept or manipulate them.
  • Server: The remote machine running the website
    🐞 Bug bounty angle: Identify server-side misconfigurations or outdated software.
  • Protocol: HTTP/HTTPS to communicate
    🐞 Bug bounty angle: Spot insecure endpoints or downgrade attacks.
  • Request: You ask the server for something
    🐞 Bug bounty angle: Modify requests to test for input validation and access control.
  • Response: The server sends it back
    🐞 Bug bounty angle: Analyze headers and content for sensitive data leaks.

📬 HTTP Requests & Responses

🟦 Request Example

GET /index.html HTTP/1.1
Host: example.com

🟩 Response Example

HTTP/1.1 200 OK
Content-Type: text/html
Set-Cookie: session=xyz123

🐞 Bug bounty angle: Inspect headers for sensitive data, cookies, and misconfigurations.

🔢 Key HTTP Methods

MethodWhat It Does🐞 Bug Bounty Angle
GETRetrieve dataCheck for sensitive data exposed via query parameters.
POSTSubmit dataTest for input validation and injection vulnerabilities.
PUTUpdate resourceLook for unauthorized access or insecure APIs.
DELETERemove resourceCheck for privilege escalation or lack of auth.
HEADLike GET, no bodyUse for stealthy probing of endpoints.

⚠️ Common HTTP Status Codes

CodeMeaning🐞 Bug Bounty Angle

302Temporary RedirectSame as above—test redirect logic.
403ForbiddenTry bypassing with headers or methods.
404Not FoundUse for fuzzing hidden endpoints.
