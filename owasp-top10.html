🛡️ OWASP Top 10

Understand the most critical web vulnerabilities. Hunt smarter, exploit cleaner.

🔐 A01: Broken Access Control

Access control failures allow attackers to act as users or admins without proper authorization.

  • Examples: IDOR, privilege escalation, bypassing role checks
  • Common causes: Missing authorization, predictable URLs

🐞 Bug bounty angle: Try accessing other users’ data. Look for endpoints that don’t enforce role or permission checks.

🔒 A02: Cryptographic Failures

Weak or missing encryption exposes sensitive data in transit or at rest.

  • Examples: No HTTPS, weak TLS, exposed secrets
  • Common causes: Misconfigured SSL, outdated crypto libraries

🐞 Bug bounty angle: Check for plaintext transmission, missing `Secure` flags, or weak TLS versions.

💉 A03: Injection

Untrusted data sent to interpreters can lead to code execution or data leaks.

  • Examples: SQLi, XSS, command injection
  • Common causes: Unsanitized input, dynamic queries

🐞 Bug bounty angle: Inject payloads into parameters. Use Burp Intruder or manual fuzzing to find vulnerable inputs.

🧠 A04: Insecure Design

Flaws in logic or architecture that attackers can exploit even if code is secure.

  • Examples: Insecure workflows, missing rate limits
  • Common causes: Lack of threat modeling, poor UX decisions

🐞 Bug bounty angle: Think like a user. Abuse flows, skip steps, or test edge cases that break logic.

⚙️ A05: Security Misconfiguration

Default settings, verbose errors, and open ports expose systems to attack.

  • Examples: Directory listing, debug mode, exposed admin panels
  • Common causes: Incomplete hardening, forgotten endpoints

🐞 Bug bounty angle: Scan for open directories, verbose error messages, and forgotten dev tools.

📦 A06: Vulnerable & Outdated Components

Using libraries or platforms with known vulnerabilities can lead to compromise.

  • Examples: CVEs in jQuery, outdated WordPress plugins
  • Common causes: No patching, legacy dependencies

🐞 Bug bounty angle: Identify tech stack. Search for known CVEs and test for exploitability.

🔑 A07: Identification & Authentication Failures

Weak login mechanisms allow attackers to impersonate users or hijack sessions.

  • Examples: Brute-forceable logins, session fixation
  • Common causes: Missing MFA, predictable tokens

🐞 Bug bounty angle: Test login endpoints for rate limits, token predictability, and session hijacking.

🧬 A08: Software & Data Integrity Failures

Trusting unverified code or data can lead to tampering or supply chain attacks.

  • Examples: Insecure deserialization, CI/CD injection
  • Common causes: No integrity checks, unsafe plugins

🐞 Bug bounty angle: Look for places where user input affects backend logic or deployment flows.

📉 A09: Security Logging & Monitoring Failures

Without proper logging, attacks go undetected and uninvestigated.

  • Examples: No logs, no alerts, delayed response
  • Common causes: Missing audit trails, poor SIEM integration

🐞 Bug bounty angle: Trigger errors and observe response. Look for lack of alerts or logging indicators.

🌐 A10: Server-Side Request Forgery (SSRF)

SSRF tricks the server into making requests to internal systems.

  • Examples: Accessing metadata services, internal APIs
  • Common causes: Unvalidated URLs, open fetch logic

🐞 Bug bounty angle: Inject internal IPs or cloud metadata URLs into fetch parameters. Look for response leaks.
